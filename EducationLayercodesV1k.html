Ansvarig part / Responsible party
Required to be filled in



The layercode begins with a field that identifies who is responsible for the information that is on the layer. The tecnical area/disciplin is a larger list than those used for file names. This responsibility should not be confused with the technical area that is implicit in the element code, for example an architect works on similar element codes as construction
Element
Required to be filled in



Byggdelar/building parts: BSAB96 Buildingparts according to Trafikverkets Strukturplatsen (which is BSAB96 plus more civil classifications). A building part is a physical part of a whole idenfitied from its primary function. Building parts are idenfitied from a functional perspective, top down

Alternativa tabeller / Alternative tables: All information that can not be structured with BSAB96, for example objects that represent non-physical phenomena: contractor/tender legalities, space usage, fire restrictions for ex. There is a therefor a demand to use other klassification systems. In alternative tables Trafikverket has Tunnel and Rock with for example rock klassifications

Landskapsinformation / terrain information: For coding of existing conditions/objects, terrain information, the following process can be used. For natural occuring conditions terrain information can be used. For built existing conditions you primarely use Byggdelar/Building parts or in second hand terrain information. Existing information should be coded as Existing (E) in status
Presentation
Required to be filled in



Presentation codes classify layers on what type of graphics they include. Graphics includes 2D/3D that describes the shape of the information but also graphics that are part of boundaries, text and measurements.

Simplifiedfor mnesomrdesmodeller (disciplin modells) the layer that has either 2D or 3D objects that represent something and therefor has a matching element are coded as 3D (EZ) or 2D (EJ) while text in the modell is Text (T-)

For drawings there are codes that help manage multiple layers while still conforming to the requirement "Color by layer"
Status
Required to be filled in



Status describes what kind of status the building part has in the production situation. It doesn't describe past the building phase, it is a description of the production situation. Trafikverket har additions to SB11s recommendation in the form of Krav, Frslag, Utan status, Stngd. If you plan to use Krav you should consult Trafikverket first. There should be a plan how to make it appararent and accessible for the contractor when used
Typ / Type
Not Required



Type is an addition from Tvrfrbindelse Sdertrn from the system design phase to SB11 on top of SB11s and Trafikverkets required fields (TDOK 2012:35 5.2.3).

It captures the user defined fields in SB11 (chapter 8) Lge (placement) Ursprung (origin of information) and adds other tables. It is multiple tables in one field of 3 codes, but there is no hierarchy in the codes. It has no link to Trafikverkets Strukturplatsen

To filter, start by typing for example "Typ VA" to see everything in that table

The tables

Typ Lge: For example road division according to BH90 Del 7

Typ Ursprung: What origin the information has, scanning, digitized etc

Typ Geoteknik Use in conjuction with Landskapsinformation -> GEO*

Typ Grundkonstruktioner:

Typ Markering: Krfltslinje/Cykelfltslinje/Kantlinje etc

Typ Projektgemensamt: Use with empty element as a way to structure drawing info

Typ Rcken: Different types av barriers

Typ Skyltar: Different types av color for signs

Typ Underbyggnad: Different types of materials for substructures

Typ VA: Different types of materials for water production results

Typ Vegetation:

Typ Vgplan: Helper classification for regulations of the road plan

Typverbyggnad: Different types of material for superstructures

Anlggningsdel / Construction part
Not required



Anlggningsdelar (constructionparts) is a system of dividing a road facility into parts that are then used to place installation components (komponenter). As a field in the layer code, it is usually used where you have one file that include multiple anlggningsdelar as objects. For example if the file is K300* (common in delomrde/area 3) and includes several constructions (lets say 341,342,346), the constructions can have different layers with the anlggningsdel in the end separating them and making them searchable
